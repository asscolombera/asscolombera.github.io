General

MenuBuilder is a windows32 application to help creating and maintaining javascript variable files belonging to Menu version 9.10 (+)
No extensive knowledge of javascript is needed to build and maintain a menu.
The used label names in front of the input fields and checkboxes correspondent as much as possible with the actual javascript variable names. For detailed explanation of the variables see: installmenus.html

Typing conventions

Colors
You can pick a color from the dropdown list, choose one with the color dialog window or enter one manually.
Colors can be any valid color name or a rgb representation in the format #RRGGBB, where R G and B are hexadecimal values between 0 and F.
Fonts
You can choose a font with the font dialog window or enter one manually.
You can specify more fonts seperated with a comma: arial, system, technical. The system will use the first font found on the users system.
Exlamation fields
Some fields, marked by (!) in front of the label accept a ! as first character in the field. When MenuBuilder finds a ! at the beginning in those fields no error checking is done and the input is not surrounded with qutations.
This is usefull for users that have good knowledge of javascript to use javascript statements in those fields instead of text.
An example is the default value of BaseHref. It uses:
!document.location.href.substring(0,document.location.href.lastIndexOf("/") +1) as default input value.
The statement calculates the root of the site from the location of the document that loads the script.
Number Input
Valid characters are: 0-9, . and -. Values between 0 and 1 can be entered as .4 or 0.4

Getting Started

Create a template/ project
When you start from scratch you might consider to create a template. MenuBuilder creates all necassay HTML and Javascript files. It looks for menu9_com.js in the specified create folder and when it does not find one it copies the one from the MenuBuilder folder.
You can create a template for a single page, vertical frames, horizontal frames or hiddenframes setup. Finally you can use the current settings of the variables to let MenuBuilder create a template based on the current variables. All other templates reset the variables. When you use menu_compat.js you need to rename it to menu9_com.js and keep a copy in the MenuBuilder folder.
Open an existing template/ project
Once you created a template you can reopen the template as project. Menubuilder asks for the main HTML file, which is either a single HTML document or a frameset document when you use frames, and the relevant javascript variable file.
MenuBuilder opens the variable file for editing and uses the HTML file for preview. Of course you can open an existing project, not created with MenuBuilder, in the same manner.
Maintain an existing template/ project
To maintain an existing variable file you can open it independently or open the project that holds the variable file.

MenuBuilder's menu options

File menu
  • Reset variable file
    Asks if you want to save the current file, closes an open project and resets all the variables to their default inititial values.
  • Open variable file
    Opens an existing variable file.
  • Save variable file
    Saves an opened variable file.
  • Save variable file as
    Saves an opened variable file under another name.
  • Create HTML template
    Lets you create an template. See Create a template
  • Open HTML project
    Opens a project. See Opening an existing template
Tools menu
  • Insert preload of menu images
    Recommended. MenuBuilder searches your variable file for images and inserts the necassary code in the function AfterBuild to preload them.
  • Insert hide/show code
    MenuBuilder inserts the necassary code in the functions BeforeFirstOpen and AfterCloseAll to hide /show embedded objects in your HTML that are outside the documents z-order.
    See: Hide form elements and objects
  • Insert current variable file in existing HTML
    MenuBuilder asks for the name of the HTML file and inserts the script loader based on the current variable file. It saves the current variable file to the same directory as the main HTML file.
View menu
  • Preview in browser
    MS IE 4+ has to be installed on your system to use this function.
  • Javascript code
    Shows the javascript code as it will be saved to a variable file.
Help menu
 
  • Using MenuBuilder
    Opens this file.
  • Menu Javascript
    Opens installmenus.html which descibes the variables and other menu issues.
  • About MenuBuilder
    Displays the version and copyright.
